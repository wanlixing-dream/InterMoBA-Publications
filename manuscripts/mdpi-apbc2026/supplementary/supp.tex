\section{Supplementary Methods}

\subsection{Detailed MoBA Mechanism}

\subsubsection{Dynamic Partition and Top-K Selection Details}

The MoBA mechanism dynamically allocates attention resources by computing an importance score for each node/edge and adjusting feature chunking via a Top-K selection strategy. The chunk size for each region is dynamically set based on an aggregated score of its node and edge features, distance, contact frequency, and centrality.

\textbf{Supplementary Eq. S1: Calculation of Dynamic Chunk Size.}
The chunk size for a batch region $b$ is determined as follows:
\begin{equation}
\text{ChunkSize}_b = \max(\text{MergedLen}_b, \text{BaseChunkSize})
\end{equation}
where $\text{BaseChunkSize}=32$ is the minimum granularity used in our experiments. $\text{MergedLen}_b$ is the length of the merged key region after fusing importance scores. The fusion score for a node $i$ is calculated as:
\begin{equation}
\text{FusionScore}_i = \text{MLP}_{\text{fuse}}\left(\left[ \text{Imp}_i, \text{ContactFreq}_i, \text{Centrality}_i \right]\right)
\end{equation}
Here, $\text{Imp}_i$ is the node importance score, $\text{ContactFreq}_i$ is the contact frequency, and $\text{Centrality}_i$ is a centrality measure. These scores are then normalized using softmax:
\begin{equation}
\text{SoftmaxScore}_i = \frac{\exp(\text{FusionScore}_i)}{\sum_j \exp(\text{FusionScore}_j)}
\end{equation}
The Top-K highest-scoring regions are selected for full-precision attention:
\begin{equation}
\text{Top-K} = \text{argsort}(\text{SoftmaxScore})[:k]
\end{equation}
where $k = \max(10, \lceil 0.05 \times N \rceil)$, and $N$ is the total number of nodes in the input graph.

\subsection{Training Hyperparameters}
The model was trained using the AdamW optimizer with an initial learning rate of $1\times10^{-4}$, a weight decay of $1\times10^{-2}$, and a batch size of 32. We employed a cosine annealing learning rate scheduler with a warm-up period of 5000 steps. Mixed-precision training (fp16) was used to accelerate computation and reduce memory footprint. All models were trained on a single NVIDIA RTX 4090 GPU for a maximum of 200 epochs, with early stopping based on the validation loss.

\subsection{Dataset Split Details}
For the time-split PDBBind benchmark, we followed the protocol established in previous works (e.g., DiffDock, Interformer). Specifically, we used the PDBBind v2016 dataset and split complexes based on their PDB release year, ensuring that all test set complexes have a release year later than 2016, while training and validation sets contain older entries. This yields 333 complexes in the test set. The full (random) split is also used for ablation studies.

\subsection{Monte Carlo Sampling Procedure}
InterMoBA generates candidate ligand poses via a Monte Carlo (MC) sampling algorithm. Starting from an initial random orientation and torsion angles of the ligand within the binding pocket, the MC simulation performs 5000 steps. At each step, a random translation, rotation, or torsion change is proposed, and the energy (negative log-likelihood of the mixture density function) is evaluated. The proposal is accepted according to the Metropolis criterion. The lowest-energy conformations among the sampled trajectory are retained as the Top-K poses (K=10 by default).
